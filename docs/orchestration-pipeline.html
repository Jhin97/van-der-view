van-der-view — Team Orchestration Pipeline

How issues, PRs, CI, and the leader Claude Code session move work from inputs to merged main

Merge gates (all must hold)

  • • `Closes #N` present and issue exists with `team:*`
  • • Changelog section filled
  • • ≥1 CI check exists and all required pass
  • • Leader's review = approve, no unresolved threads
  • • `leader:auto-merge` OR human reviewer also approved
  • • `leader:hold` not set · `mergeable: MERGEABLE`

Label semantics

  • • `team:*` — routing (add new teams freely)
  • • `type:project-plan` — leader runs /ingest-project-plan on merge
  • • `priority:p0/p1/p2` — surfaces in the daily digest
  • • `status:*` — leader-managed lifecycle state
  • • `leader:auto-merge` — opt-in: leader may merge solo
  • • `leader:hold` — opt-out: leader comments only

Stack-agnostic test policy

  • • Leader never executes test commands itself
  • • Source of truth: GitHub status checks on the PR
  • • Re-run via `gh workflow run` when needed
  • • No checks configured → no auto-merge, ever
  • • Survives stack changes (Vitest → Playwright, Node → Python)
  • • Adding/changing CI is a `team:infra` issue

Post-merge bookkeeping

  • • Append PR's Changelog line to `## [Unreleased]` on main
  • • Separate commit, not in the PR (avoids file conflicts)
  • • Close linked issue with `Merged in #<PR> (<sha>).`
  • • Comment on issues listing this one in `Depends on:`
  • • Co-Authored-By the leader bot identity

Self-throttling cadence

  • • Open PRs with PENDING checks → 90s wake (cache-warm)
  • • Open PRs, CI resolved → 270s wake
  • • Idle (no open PRs) → 1800s wake
  • • Driven by ScheduleWakeup inside `/loop`
  • • Avoids burning context cache on no-op cycles

Member workflow (humans)

  • • Pick an issue with your team's label
  • • Branch `<team>/<n>-<slug>`
  • • Local Claude Code: /plan → Explore → implement → self-review
  • • Open PR with `Closes #N` and filled Changelog section
  • • Add `leader:auto-merge` to skip the human-approver step
  • • Leader handles the rest
